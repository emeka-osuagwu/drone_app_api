Usage

Example POST request to the register route 
          
            POST https://sweetemoji.herokuapp.com/auth/register

            Request Body:
            {
                'username' : 'johndoe',
                'password' : 'password',
            }
Upon successful register, a Json object is return containing the status code, the token generated, the token expiry and the success message. A response header containing the 'Authorization' token is is also set. This token is required, as an 'Authorization' header to access all protected routes. 
         
            Response Body:
                {
                  "status": 200,
                  "massage": "Registration Successful"
                }
Example POST request to the login route 
          
            POST https://sweetemoji.herokuapp.com/auth/login

            Request Body:
            {
            'username' : 'johndoe',
            'password' : 'password',
            }
Upon successful login, a Json object is return containing the status code, the token generated, the token expiry and the success message. A response header containing the 'Authorization' token is is also set. This token is required, as an 'Authorization' header to access all protected routes. 
         
            Response Body:
            {
                'token'  : 'SomeRandomToken'
            }
GET request to the logout route 
        
         GET https://sweetemoji.herokuapp.com/auth/logout

          Request Body:
          {
            'token' : 'SomeRandomToken'
          }
Upon successful fulfil of request, the token is unset and a json object returned containing the message. 
       
        Response Body:
        {
            'status':  200,
            'message': 'token unset'
        }
TO save an Emoji, a POST request has to be made to 
https://sweetemoji.herokuapp.com/emojis
Like so: 
        
         POST https://sweetemoji.herokuapp.com/emojis
         Request Body:
         {
            'tag'   : 'tags',
            'title' : 'someName',
            'image' : 'happyface.png',
         }
        
     
 Note that the 'Authorization' header should contain the token
The response received from that request will be of this form 
     
        Response Body:
        {
            'status'  : 201,
            'message' : 'Record created'
        }
To get all emojis, a GET request should be sent to 
         
             https://sweetemoji.herokuapp.com/emojis
This route is not protected and does not need an authorization header. Example: 
        
            GET https://sweetemoji.herokuapp.com/emojis
        
      
        
         Response Body:

         [
            {
                'id'            : 1,
                'tag'           : 'smile, face, happy',
                'title'         : 'smile',
                'image'         : 'happyface.png',
            },

            {
                'id'            : 2,
                'tag'           : 'sad, face, cry',
                'title'         : 'sad',
                'image'         : 'sadface.png',
            }
         ]
To edit an emoji send a PUT request to 
        
            https://sweetemoji.herokuapp.com/emojis/{id}
Example: 
         
           PUT https://sweetemoji.herokuapp.com/emojis/1

           Request Body
            {
                'id'            : 1,
                'tag'           : 'smile, face, happy',
                'title'         : 'smile',
                'image'         : 'happyface.png',
            },
All arguments, except the id, are optional. 
          
            Response Body:

            {
                'status'  :  200,
                'message' : 'Record modified'
            }
To delete an Emoji, send a DELETE request to 
           
                https://sweetemoji.herokuapp.com/emojis/{id}
Example: 
            
            DELETE https://sweetemoji.herokuapp.com/emojis/1

            Request Body:
            {
                'id' : 1
            }
The reponse will be like so: 
            
                Response Body:
                {
                    'status'  :  200,
                    'message' : 'Record deleted'
                }
